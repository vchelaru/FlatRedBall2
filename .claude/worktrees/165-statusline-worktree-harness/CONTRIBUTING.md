# Contributing to FlatRedBall2

First off, thank you for considering contributing to FlatRedBall2! 

## How to Contribute

### Reporting Bugs
If you find a bug, please open an issue and include:
- A clear description of the problem.
- Steps to reproduce the bug.
- Which backend you are using (MonoGame or Kni).

### Submitting Pull Requests
1. **Fork** the repository.
2. **Branch** off of `main` for your work.
3. **Commit** your changes, keeping messages clear and descriptive.
4. **Test** your changes locally (ensure `dotnet test` passes).
5. **Open a PR** against the `main` branch.

Please ensure any new features or bug fixes are accompanied by tests in the `tests/` directory when appropriate.

## Code Style
Please try to match the existing formatting and code style in the repository. We use standard C# conventions.
